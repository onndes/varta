## 💡 ЯК ДОДАТИ НОВІ ТЕСТИ

### 1. Створити файл:

```typescript
// tests/services/myService.test.ts
import { describe, it, expect, beforeEach, afterEach } from 'vitest';
import { db } from '@/db/db';
import { myFunction } from '@/services/myService';

beforeEach(async () => {
  await db.delete();
  await db.open();
});

afterEach(async () => {
  await db.delete();
});

describe('myService', () => {
  describe('myFunction', () => {
    it('повинен робити щось', () => {
      const result = myFunction();
      expect(result).toBe(true);
    });
  });
});
```

### 2. Запустити:

```bash
npm test myService
```

## ✅ ПЕРЕВАГИ

### Що дають тести:

1. **Швидка перевірка логіки** ⚡
   - Не потрібно запускати весь додаток
   - Перевірка за секунди

2. **Автоматичне виявлення багів** 🐛
   - Тести покажуть що зламалось
   - Показують де саме проблема

3. **Впевненість при рефакторингу** 💪
   - Змінюй код без страху
   - Тести гарантують що логіка працює

4. **Документація коду** 📚
   - Тести показують як використовувати функції
   - Приклади використання

5. **Швидка розробка** 🚀
   - TDD підхід (Test Driven Development)
   - Пиши тест → пиши код → тест проходить
